These days GIT is almost used by every tech companies. So, it' worth learning.

    • Basic commends

    In GIT, some commands are used offend by developers. Such as:

  • git add .
  • git commit -m "commit message"
  • git push

    • Lemme explain these terms.

    git add will stage the changes.

    git commit will commit the staged changes.

    And finally, git push will upload the code to remote directory.

    Branching

    Branching is a very interesting topic. Branches are like multi dimension that allow developers work on the same project but without affecting the each others code. Confusing !?. Lets take a look at an example of a branch.

    You are working on full stack web project. You are a frontend developer and your friend Elon Musk is back developer. If you both work without branches you may encounter some problem during the development. Thats hare git branches come to handy.

    Just run.

    git branch front-end git checkout front-end

    Tadaaa, now you have a copy of the project that only for you. All you have to make the changes in your front-end branch and after everything finished just merge your frond-end code with the main branch where the final codes will be.

    How to use Github

    To use Github you have to create an account. Go and create one first.

    After that, create a new repository and copy the url and set the repository as the remote origin of your local project by running this command in your terminal.

    git remote origin -u main

    Now if you push your code it directly goes to Github repository.
